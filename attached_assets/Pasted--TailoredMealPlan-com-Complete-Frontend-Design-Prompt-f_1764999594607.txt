# TailoredMealPlan.com - Complete Frontend Design Prompt for Replit

## Project Overview

Design a complete frontend for **TailoredMealPlan.com** - an AI-powered personalized meal planning SaaS application. This is a **design-only phase** - no backend functionality required. Use placeholder data, mock states, and static content throughout.

**Tech Stack for Design:**
- React + Vite (or Next.js)
- Tailwind CSS
- Lucide React icons
- Framer Motion for animations
- React Router for navigation

---

## Brand Identity

### Brand Name
**TailoredMealPlan.com**

### Tagline
"AI-Powered Personalized Nutrition for Everyone"

### Color Palette
```css
/* Primary - Nature/Health inspired greens */
--primary: #2D5A3D;
--primary-light: #4A7C5C;
--primary-dark: #1E3D2A;

/* Accent - Warm, appetizing gold */
--accent: #E8A54B;

/* Backgrounds */

--bg-sage: #E8EDE6;
--white: #FFFFFF;

--text-muted: #5A6B5E;

/* B2B specific blues */

--b2b-accent: #3D7CC9;

/* Status colors */
--success: #22C55E;
--warning: #F59E0B;
--error: #EF4444;
```

### Typography
- **Headings:** Playfair Display (serif) - elegant, trustworthy
- **Body:** DM Sans (sans-serif) - clean, readable
- **Accent/Numbers:** Space Grotesk (for stats, prices)

### Design Tone
- Clean, modern, health-focused
- Warm and welcoming (not clinical)
- Professional but approachable
- Food imagery should look appetizing and fresh
- Use subtle food/leaf patterns as decorative elements

### 8. IMPORTANT DESIGN NOTES

1. **No Backend Required:** All forms should show static success states. No actual data processing.

2. **Use Placeholder Images:** Use Unsplash or placeholder services for food images. Suggest searches: "healthy meal", "salad bowl", "grilled chicken", "breakfast oatmeal"

3. **Consistent Spacing:** Use 8px grid system. Spacing: 8, 16, 24, 32, 48, 64px

4. **Accessibility:** Proper contrast ratios, focus states, alt text for images

5. **Animations:** Subtle hover effects, page transitions, loading states. Don't overdo it.

6. **Mobile First:** Design for mobile, then scale up to desktop

7. **Error States:** Design empty states, error messages, 404 page

8. **Dark Mode (Optional):** If time permits, add dark mode toggle
